O que é o Claude Code

Um agente de programação que roda direto no seu terminal e entende todo o seu projeto.

Em resumo: o Claude Code é como ter um programador sênior dentro do terminal. Você conversa em linguagem natural, ele lê seus arquivos, faz alterações, roda comandos e interage com o Git — tudo com sua aprovação.

O que ele pode fazer
Capacidades principais
CapacidadeExemplo prático
Ler o projeto inteiro"Explica como esse projeto está estruturado"
Editar arquivos"Adiciona animação de fade nesse botão"
Rodar comandosExecuta git, npm, scripts automaticamente
Fazer commits e push"Commita as mudanças com uma boa mensagem"
Revisar código"O que pode ser melhorado nesse arquivo?"
Criar arquivos"Cria um README detalhado para esse projeto"
Resolver bugs"Esse erro está acontecendo, me ajuda a resolver"
Como funciona na prática
O fluxo básico
Você entra na pasta do projeto, digita claude e começa a conversar. O Claude Code pede permissão antes de fazer qualquer alteração nos seus arquivos ou rodar comandos.

Diferença do chat: no claude.ai você cola código manualmente. No Claude Code ele navega e edita o projeto inteiro sozinho, com contexto completo de todos os arquivos.

Instalação no macOS

O Claude Code é instalado via npm e requer Node.js 18 ou superior.

Passo 1
Verificar Node.js
%node --version
Precisa ser v18 ou superior. Se não tiver: brew install node
Passo 2
Instalar o Claude Code
%npm install -g @anthropic-ai/claude-code
Instala globalmente via npm
Passo 3
Fazer login com sua conta Anthropic
%claude auth login
Abre o navegador para autenticar com sua conta claude.ai
Uma janela do navegador abre — faça login com a mesma conta do claude.ai, copie o código e cole no terminal.

Se preferir usar via API Key em vez de assinatura, use claude auth login --console e informe sua chave da Anthropic Console.

Passo 4
Verificar instalação
%claude --version
Mostra a versão instalada
%claude auth status
Confirma que está autenticado
Atualização
Manter sempre atualizado
%claude update
Atualiza para a última versão

Permissões
Como o Claude Code pede autorização
Antes de editar arquivos ou rodar comandos, o Claude Code sempre mostra o que vai fazer e pede confirmação:
exemplo de permissão
⚡ Edit file: index.html
Adicionar classe 'fade-in' ao elemento hero
[Y]es [N]o [A]lways [S]kip

Pressione Y para aprovar, N para recusar, A para aprovar sempre esse tipo de ação, ou S para pular.

Claude Code + GitHub

O Claude Code entende Git nativamente — você pede em português e ele executa os comandos.

Como funciona: você não precisa lembrar os comandos Git. Basta descrever o que quer fazer. O Claude Code interpreta, executa os comandos corretos e mostra o que foi feito.

Operações Git por conversa
O que você pode pedir
commit
"Commita as mudanças que fiz com uma boa mensagem descritiva"
push
"Sobe essas alterações para o GitHub"
branch
"Cria uma branch chamada melhoria-animacoes e muda para ela"
status
"O que mudou no projeto desde o último commit?"
histórico
"Mostra os últimos 5 commits com o que foi alterado em cada um"
desfazer
"Desfaz a última alteração no arquivo style.css"
Configuração recomendada
Arquivo CLAUDE.md — a memória do projeto
O arquivo CLAUDE.md na raiz do projeto funciona como instruções permanentes para o Claude Code. Ele lê esse arquivo sempre que inicia uma sessão no projeto.
peça para o Claude Code criar
"Cria um arquivo CLAUDE.md com as instruções desse projeto: nome, tecnologias usadas, e que o repositório no GitHub é luizmariojs/nome-do-repo"
%/init
Comando slash que gera o CLAUDE.md automaticamente

Com o CLAUDE.md configurado, o Claude Code sabe o contexto do projeto sem você precisar explicar toda vez. Inclua linguagem, frameworks, convenções e informações do repositório GitHub.

Subindo o guia Git para o GitHub

Passo a passo para pegar o guia que criamos e publicar no GitHub usando o Claude Code.

1
Salve o guia como arquivo HTML
No chat do claude.ai, peça para salvar o guia como arquivo. Depois salve em uma pasta no Mac, por exemplo ~/projetos/guia-git.
2
Crie o repositório no GitHub (vazio)
No GitHub: botão New → nome guia-git-macos → Create repository. Não marque nenhuma opção.
3
Entre na pasta e abra o Claude Code
%cd ~/projetos/guia-git
%claude
4
Peça para o Claude Code publicar no GitHub
você digita no Claude Code
"Inicia o Git nessa pasta, conecta ao repositório git@github.com:luizmariojs/guia-git-macos.git e publica tudo no GitHub com a mensagem 'primeiro commit: guia de git para macOS'"
O Claude Code vai executar o git init, git add, git commit e git push — pedindo sua confirmação em cada etapa.
5
Confirme que funcionou
você digita
"Mostra o status do repositório Git"

Projeto publicado! O Claude Code cuidou de todos os comandos Git para você.

Fazendo melhorias com Claude Code

O fluxo completo para melhorar o guia e publicar as mudanças no GitHub.

Fluxo de trabalho
Como funciona o ciclo de melhoria
1
Entre no projeto e abra o Claude Code
%cd ~/projetos/guia-git && claude
2
Descreva a melhoria que quer fazer
exemplos do que você pode pedir

"Adiciona exemplos práticos com outputs reais dos comandos"
"Corrige qualquer erro de português no guia"
3
Revise e aprove as mudanças
O Claude Code vai mostrar exatamente o que vai alterar no arquivo antes de salvar. Você pode aceitar, recusar ou pedir ajustes.
4
Peça para commitar e publicar
você digita
"Commita essas melhorias com uma mensagem descritiva e sobe para o GitHub"
O Claude Code cria a mensagem de commit, faz o commit e o push — tudo em uma conversa.

Melhoria publicada no GitHub! Você pode repetir esse ciclo quantas vezes quiser.

Boas práticas
Dicas para trabalhar bem com o Claude Code

Uma coisa por vez: peça uma melhoria de cada vez. Mudanças menores são mais fáceis de revisar e de desfazer se necessário.

Descreva o resultado: em vez de "muda o CSS", diga "quero que os títulos fiquem em verde e com mais espaço". O Claude Code interpreta melhor quando você descreve o que quer ver.

Peça uma branch: para mudanças grandes, peça "cria uma branch chamada nova-funcionalidade antes de editar". Assim o main fica seguro.

Dicas de uso

Como tirar mais proveito do Claude Code no dia a dia.

Prompts eficientes
Como pedir coisas ao Claude Code

Evite: "melhora o projeto" — muito vago, o Claude Code não sabe por onde começar.

Prefira: "no arquivo index.html, melhora a seção do header adicionando uma barra de navegação com os links: Início, Sobre e Contato" — específico e com contexto.

para entender o projeto
"Lê todos os arquivos e me dá um resumo do que esse projeto faz"
para uma mudança visual
"No arquivo style.css, altera o esquema de cores para usar azul escuro (#1a2744) como cor primária"
para revisão
"Revisa o código inteiro e me diz o que pode ser melhorado sem fazer nada ainda"
para documentação
"Cria um README.md profissional em português para esse projeto"
Fluxo com branches
Trabalhar com segurança
fluxo recomendado para mudanças grandes
"Cria uma branch chamada redesign-header, faz as alterações e depois me mostra um resumo antes de commitar"

Pedindo para trabalhar em uma branch, o main fica seguro. Se não gostar do resultado, é só deletar a branch.
