Instalação no macOS

Três formas de instalar o Git. A recomendada para desenvolvedores é via Homebrew.

Opção 1 — Recomendada
Via Homebrew Preferida
O Homebrew é o gerenciador de pacotes mais popular do macOS. Se ainda não tiver instalado:
%/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Instala o Homebrew
%brew install git
Instala o Git via Homebrew
Opção 2
Via Xcode Command Line Tools
%xcode-select --install
Instala as ferramentas de linha de comando da Apple
Verificação
Confirmar instalação
%git --version
Esperado: git version 2.x.x

Configuração inicial

Configure sua identidade antes de fazer qualquer commit. Salvo em ~/.gitconfig.

Obrigatório
Identidade
%git config --global user.name "Seu Nome"
Nome que aparece nos commits
%git config --global user.email "seu@email.com"
Use o mesmo email do GitHub
Recomendado
Branch padrão e editor
%git config --global init.defaultBranch main
Define "main" como branch padrão
%git config --global core.editor "code --wait"
Define VS Code como editor padrão
%git config --global core.autocrlf input
Evita problemas com quebras de linha no macOS
SSH — GitHub
Autenticação via chave SSH Mais segura
%ssh-keygen -t ed25519 -C "seu@email.com"
Gera par de chaves
%eval "$(ssh-agent -s)"
Inicia o agente SSH
%ssh-add --apple-use-keychain ~/.ssh/id_ed25519
Adiciona chave ao Keychain do macOS
%pbcopy < ~/.ssh/id_ed25519.pub
Copia a chave pública — cole em GitHub > Settings > SSH Keys
%ssh -T git@github.com
Testa a conexão SSH com o GitHub

Resposta esperada: Hi usuario! You've successfully authenticated, but GitHub does not provide shell access. — isso é normal e significa que funcionou!

Fluxos para criar um repositório

Escolha o fluxo de acordo com como você vai começar o projeto.

Fluxo A — GitHub primeiro Recomendado
Cria o repositório no GitHub e clona para o Mac. O mais simples, sem conflitos.
Fluxo B — Terminal primeiro
Inicia o Git na pasta local e conecta ao GitHub depois. Útil quando o projeto já existe.
Fluxo C — 100% terminal
Cria o repositório no GitHub via CLI sem abrir o navegador.

Quando usar: sempre que for começar um projeto do zero. É o fluxo mais limpo pois evita conflitos entre arquivos locais e remotos.

Atenção: ao criar o repositório no GitHub, não marque nenhuma opção de README, .gitignore ou licença — deixe o repositório totalmente vazio.

1
Crie o repositório no GitHub (vazio)
Acesse github.com → botão New → dê um nome → clique em Create repository sem marcar nenhuma opção.
2
Clone o repositório para o Mac
%git clone git@github.com:usuario/nome-do-repo.git
Isso cria a pasta do projeto já conectada ao GitHub
3
Entre na pasta e adicione seus arquivos
%cd nome-do-repo
Entre na pasta clonada e copie seus arquivos para cá
4
Publique no GitHub
%git add .
%git commit -m "primeiro commit"
%git push
Como clonou, o push já sabe o destino — sem precisar de -u origin main

Projeto publicado! A partir de agora o ciclo é sempre: git add . → git commit -m "msg" → git push

Quando usar: quando o projeto já existe na sua máquina e você quer versioná-lo no GitHub agora.

Atenção: ao criar o repositório no GitHub, não marque nenhuma opção de README, .gitignore ou licença — repositório completamente vazio.

1
Entre na pasta do projeto e inicie o Git
%cd ~/caminho/para/seu-projeto
%git init
Cria o repositório Git local na pasta
2
Crie o repositório no GitHub (vazio)
Acesse github.com → botão New → dê um nome → clique em Create repository sem marcar nenhuma opção.
3
Conecte o repositório local ao GitHub
%git remote add origin git@github.com:usuario/nome-do-repo.git
4
Adicione os arquivos, faça o commit e publique
%git add .
%git commit -m "primeiro commit"
%git push -u origin main
O -u só é necessário no primeiro push — depois basta git push

Projeto publicado! Nas próximas vezes: git add . → git commit -m "msg" → git push

Quando usar: quando você quer fazer tudo pelo terminal sem abrir o navegador. Requer o GitHub CLI (gh) instalado.

1
Instale o GitHub CLI (se ainda não tiver)
%brew install gh && gh auth login
Instala e autentica com o GitHub
2
Entre na pasta do projeto e inicie o Git
%cd ~/caminho/para/seu-projeto
%git init
3
Faça o primeiro commit
%git add .
%git commit -m "primeiro commit"
4
Crie o repositório no GitHub e publique — tudo em um comando
%gh repo create nome-do-repo --public --source=. --remote=origin --push
Cria o repo no GitHub, conecta e faz o push em um só comando
Troque --public por --private para repositório privado.

Projeto criado e publicado sem abrir o navegador! Nas próximas vezes: git add . → git commit -m "msg" → git push

Resumo comparativo
Qual fluxo escolher?
Situação Fluxo recomendado
Projeto novo do zero Fluxo A — GitHub primeiro (mais simples, sem conflitos)
Projeto já existe na máquina Fluxo B — Terminal primeiro
Quer tudo pelo terminal Fluxo C — 100% terminal com gh CLI
Contribuir em projeto de outros Fork + Clone (ver seção Colaboração)

Fundamentos do Git

O ciclo básico de trabalho: adicionar arquivos, commitar e repetir.

Conceito: Git rastreia alterações em 3 áreas — working directory (seus arquivos), staging area (o que vai no próximo commit) e repository (histórico confirmado).

Status e diff
Verificar o estado do repositório
%git status
Mostra arquivos modificados e staged
%git diff
Mostra diferenças em arquivos modificados
%git diff --staged
Mostra diferenças em arquivos staged
Staging
Adicionar arquivos ao stage
%git add arquivo.txt
Adiciona um arquivo específico
%git add .
Adiciona todos os arquivos modificados
%git add -p
Adiciona em partes (interativo)
%git rm --cached arquivo.txt
Remove arquivo do stage (mantém local)
Commit
Salvar alterações no histórico
%git commit -m "mensagem descritiva"
Salva com mensagem em uma linha
%git commit -am "mensagem"
Faz add e commit de um só comando
%git commit --amend -m "nova mensagem"
Edita o último commit
Ignorar arquivos
Criar .gitignore
%echo ".DS_Store" >> .gitignore
Ignora arquivos do macOS
%echo "node_modules/" >> .gitignore
Ignora pasta de dependências

Use gitignore.io para gerar .gitignore adequado para sua stack.

Branches

Branches permitem trabalhar em funcionalidades separadas sem afetar o código principal.

Fluxo recomendado: mantenha main sempre estável. Crie uma branch por funcionalidade, faça commits lá e depois mescle com main.

Criar e navegar
Gerenciar branches locais
%git branch
Lista branches locais
%git branch -a
Lista todas as branches (local e remota)
%git switch -c nova-branch
Cria e muda para nova branch
%git switch nome-da-branch
Muda para uma branch existente
%git branch -d nome-da-branch
Deleta branch (se já mergeada)
%git branch -D nome-da-branch
Força a deleção da branch
Merge e Rebase
Mesclar branches
%git switch main && git merge nome-da-branch
Mescla branch na main
%git rebase main
Reaplica commits em cima de main

Atenção: nunca faça rebase em branches que outros colaboradores já usam.

Stash
Guardar mudanças temporariamente
%git stash
Guarda mudanças para depois
%git stash pop
Recupera as mudanças guardadas
%git stash list
Lista todos os stashes

Remoto & GitHub

Sincronize seu repositório local com o GitHub para colaborar e fazer backup.

Configurar remoto
Gerenciar conexão com o GitHub
%git remote add origin git@github.com:usuario/repo.git
Conecta a um repositório remoto
%git remote -v
Lista remotes configurados
%git remote set-url origin nova-url
Muda a URL do remoto
Push e Pull
Enviar e receber mudanças
%git push -u origin main
Primeiro push, conecta branch local com remota
%git push
Envia commits para o GitHub
%git pull
Baixa e mescla mudanças do remoto
%git fetch origin
Baixa mudanças sem mesclar
%git push --force
Force push (use com cuidado!)
GitHub CLI
GitHub no terminal Bônus
%brew install gh && gh auth login
Instala e autentica GitHub CLI
%gh repo create nome --public --source=. --push
Cria repo e publica em um comando
%gh pr create
Cria Pull Request pelo terminal

Histórico de commits

Navegue, pesquise e entenda o que mudou no projeto ao longo do tempo.

Log
Ver histórico
%git log --oneline
Lista commits em uma linha cada
%git log --oneline --graph --all
Mostra árvore de commits e branches
%git log --author="Nome" --since="2024-01-01"
Filtra por autor e data
%git show abc1234
Mostra detalhes de um commit
%git blame arquivo.txt
Mostra quem modificou cada linha
Tags
Marcar versões
%git tag -a v1.0.0 -m "versão 1.0.0"
Cria tag anotada (recomendado)
%git push origin --tags
Envia tags para GitHub

Desfazer alterações

Corrija erros em diferentes estágios do fluxo.

Regra de ouro: nunca reescreva histórico de commits já enviados ao GitHub e usados por outros. Para isso, use revert.

Working directory
Desfazer mudanças não commitadas
%git restore arquivo.txt
Descarta mudanças locais do arquivo
%git restore --staged arquivo.txt
Remove arquivo do stage
Commits locais
Desfazer commits não publicados
%git reset --soft HEAD~1
Desfaz commit, mantém mudanças staged
%git reset --hard HEAD~1
Desfaz commit e descarta mudanças
Commits publicados
Reverter sem reescrever histórico Seguro
%git revert HEAD
Cria novo commit que desfaz o anterior
Emergência
Recuperar commits "perdidos"
%git reflog
Mostra histórico de movimentos do HEAD
%git switch -c recuperado abc1234
Recupera commit específico em nova branch

O reflog guarda tudo por 90 dias. Se fez um reset --hard errado, o commit ainda está lá!

Colaboração no GitHub

Fluxo de trabalho em equipe com forks, Pull Requests e resolução de conflitos.

Fork & PR
Contribuir para projetos externos
%gh repo fork usuario/repo --clone
Faz fork e clona para sua máquina
%git switch -c minha-contribuicao
Cria branch para sua feature
%git push origin minha-contribuicao
Envia branch para seu fork
%gh pr create --title "Descrição da contribuição"
Abre Pull Request para o projeto original
Conflitos
Resolver conflitos de merge
<<<<<<< HEAD
seu código aqui
=======
código do remoto aqui
>>>>>>> origin/main
%git add arquivo-resolvido.txt && git commit -m "resolve conflito"
Resolve o conflito e faz commit
%git merge --abort
Cancela o merge se não conseguir resolver
